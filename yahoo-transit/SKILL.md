# Yahoo!乗換案内スキル

Yahoo!乗換案内を `web_fetch` で取得して、乗り換えルートを返す。

## URL構造

```text
https://transit.yahoo.co.jp/search/result?from={出発地}&to={到着地}&y={年}&m={月}&d={日}&hh={時}&m1={分の十の位}&m2={分の一の位}&type={検索タイプ}&ticket=ic&al=1&shin=1&ex=1&hb=1&lb=1&sr=1
```

## パラメータ

- `from`: 出発地（駅名/バス停名）
- `to`: 到着地（駅名/バス停名）
- `y`: 年（例: `2026`）
- `m`: 月（例: `02`）
- `d`: 日（例: `24`）
- `hh`: 時（24h）
- `m1`: 分の十の位
- `m2`: 分の一の位
- `type`: `1=出発時刻指定`, `4=到着時刻指定`
- `ticket`: `ic`（IC運賃優先）
- `al`: `1`（航空路線を含む）
- `shin`: `1`（新幹線を含む）
- `ex`: `1`（有料特急を含む）
- `hb`: `1`（高速バスを含む）
- `lb`: `1`（路線バスを含む）
- `sr`: `1`（結果表示）

### ソートオプション（URLに `&s=` を追加）
- `s=0`: 到着時刻順（デフォルト）
- `s=1`: 乗換回数順
- `s=2`: 料金の安い順

## 使い方

1. ユーザーの出発地・到着地・日時（出発/到着基準）を確認
2. URLを組み立てて `web_fetch`（`extractMode: text`）で取得
3. 結果テキストから「ルート1」「ルート2」などの単位で分割して要約
4. 上位3件を優先して返す（必要なら次ページ相当も追加取得）

## 注意点

- **駅名の曖昧さ**: 同名駅がある場合は `駅名(都道府県)` を使う（例: `栄(愛知県)`）
- **地名とバス停の混同**: 地名が意図しない施設名にマッチすることがあるため、必要に応じて最寄り駅へ言い換える
- **from/toはURLエンコード必須**
- **結果は改行なしの連続テキスト**で返ることがあるため、ルート見出しキーワードで分割して整形する

## 例

```text
# 東京→横浜、平日 09:00 出発
web_fetch: https://transit.yahoo.co.jp/search/result?from=%E6%9D%B1%E4%BA%AC&to=%E6%A8%AA%E6%B5%9C&y=2026&m=02&d=24&hh=09&m1=0&m2=0&type=1&ticket=ic&al=1&shin=1&ex=1&hb=1&lb=1&sr=1
```
